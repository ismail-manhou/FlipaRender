# FlipaRender v10

أداة رندر أنيميشن احترافية تعمل على **Android (عبر Termux)**، **Linux**، و **Windows**. تحوّل سلسلة صور (فريمات) إلى فيديو أو GIF، مع رندر آلي ذكي، ذكاء اصطناعي للفريمات البينية، ودعم كامل للصوت — مع الحفاظ على خفة الأداء على الهواتف.

---

## التثبيت

### المتطلبات
- Python 3.10 أو أحدث
- FFmpeg (مطلوب لتصدير MP4؛ تصدير GIF يعمل بدونه)
- مكتبات بايثون: `Pillow`, `numpy`

### Termux (Android)
```bash
pkg install python ffmpeg
pip install pillow numpy
```

### Linux
```bash
sudo apt install python3 python3-pip ffmpeg
pip install pillow numpy --break-system-packages
```

### Windows
1. ثبّت Python من [python.org](https://python.org)
2. ثبّت FFmpeg من [ffmpeg.org](https://ffmpeg.org) (أو `winget install ffmpeg`)
3. ```
   pip install pillow numpy
   ```

## بنية المشروع المتوقَّعة

ضع كل مشاهدك في مجلد واحد، كل مشهد في مجلد فرعي يحتوي صوره مرقّمة:

```
MyProject/
├── Walk/
│   ├── frame_00.png
│   ├── frame_01.png
│   └── frame_02.png
├── Jump/
│   ├── frame_00.png
│   └── frame_01.png
├── audio/                      ← اختياري
│   └── soundtrack.mp3
└── timing.txt                  ← اختياري (Exposure Sheet يدوي)
```

يدعم البرنامج أيضاً **Compound Scenes** (مشاهد بطبقات متعددة) — كل ذلك يعمل كما كان في الإصدارات السابقة.

## التشغيل

### الرندر اليدوي (تفاعلي)
```bash
python3 main.py
```
يسألك خطوة بخطوة: مسار المشروع → المشاهد → FPS → الجودة → الفلتر → AI → الصوت → إلخ. كل سؤال له قيمة افتراضية معروضة بين `[ ]` — اضغط Enter لقبولها.

### الرندر التلقائي (Watch Mode)
```bash
python3 main.py --watch /path/to/MyProject
```
يراقب المجلد باستمرار، ويُعيد الرندر تلقائياً عند أي تعديل (إضافة/حذف/تغيير صورة)، حتى تُوقفه بـ **Ctrl+C**. لا يطرح أي سؤال — يستخدم إعدادات `project.flipa` المحفوظة لهذا المشروع لو وُجدت، وإلا إعدادات افتراضية ثابتة.

> كلا الوضعين يستخدمان **نفس محرك الرندر** (`core/render_engine.py`) — أي تحسين يُضاف للمحرك يعمل في الوضعين معاً.

## الميزات

### 1. Memory Optimizer (رندر بالدفعات)
يعالج الفريمات على دفعات (`render_chunk_size`، افتراضياً 50 فريماً) بدل تحميل كل الصور في الذاكرة دفعة واحدة. يسمح برندر آلاف الفريمات على أجهزة ذات RAM محدودة. حجم الدفعة قابل للتخصيص ويُقترح تلقائياً حسب ذاكرة جهازك.

### 2. Smart FPS Detection
يحلّل صفحة التوقيت (Exposure Sheet) تلقائياً، ويقترح FPS مناسباً حسب كثافة الحركة:
- مشاهد بـ Holds كثيرة (حركة بطيئة) → يقترح **12 fps**
- حركة سريعة → يقترح **18–24 fps**

يمكنك قبول الاقتراح أو إدخال قيمة مختلفة يدوياً.

### 3. Motion Blur
طمس حركة اختياري بين الفريمات المتتالية، يُحسّن سلاسة الحركة خاصة عند معدلات FPS منخفضة (12fps). الشدة قابلة للتحكم (0.0 إلى 0.8)، والبرنامج يقترح شدة مناسبة حسب الـ FPS المختار.

### 4. Scene Preview Grid
قبل بدء الرندر، يعرض قائمة منظمة بكل المشاهد المكتشفة (الاسم، عدد الفريمات، نوع المشهد)، مع خيار توليد صورة Contact Sheet (شبكة صور مصغّرة) لمعاينة بصرية سريعة.

### 5. AI In-betweening + AI Auto Mode
يولّد فريمات إضافية بين الفريمات الموجودة لحركة أنعم، بأربعة أوضاع:

| الوضع | الاستخدام الأمثل |
|---|---|
| `linear` | مشاهد شبه ثابتة — أسرع |
| `smart` | حركة معتدلة محصورة في جزء من الكادر |
| `hybrid` | حركة قوية محصورة + خلفية ثابتة (الأكثر توازناً) |
| `optical_flow` | حركة كبيرة منتشرة في كل الكادر — أعلى جودة |

**AI Auto Mode**: في الرندر اليدوي، يحلّل البرنامج فريمات كل مشهد تلقائياً (شدة الحركة + نسبة المساحة المتأثرة) ويقترح أفضل وضع مع شرح السبب — يمكنك قبوله أو اختيار وضع آخر يدوياً.

> ملاحظة: في **Watch Mode**، لا يحدث تحليل تلقائي لكل دورة رندر — الوضع المحفوظ في `project.flipa` (`ai_mode`) يُستخدم كما هو حرفياً، لأن لا يوجد تفاعل بشري ليُسأل.

### 6. Audio Support
دعم كامل للصوت من مجلد `audio/` داخل مشروعك (MP3 / WAV / OGG):
- **نطاق التطبيق**: الصوت على الفيديو المدموج النهائي فقط، أو على كل مشهد منفرد + المدموج
- **أنماط المزامنة**:
  - `match_video` — الصوت يُقصّ ليطابق طول الفيديو
  - `match_audio` — الفيديو يُمدَّد (تكرار آخر فريم) ليطابق طول الصوت كاملاً
  - `audio_loop` — الصوت يتكرر في حلقة حتى نهاية الفيديو

### 7. Project File System (`project.flipa`)
كل مشروع يحفظ إعداداته تلقائياً في `projects/<اسم_المشروع>.flipa` (ملف JSON) بعد أي رندر ناجح. في التشغيل التالي:
- يُعرض عليك تحميل الإعدادات المحفوظة
- اختيار **"استخدام هذه الإعدادات مباشرة"** يتخطى كل الأسئلة بالكامل
- أو يمكنك مراجعة كل إعداد يدوياً، مع المحفوظ سابقاً كقيمة مقترحة

هذا الملف هو نفسه الذي يقرأه **Watch Mode** لتحديد إعداداته.

### 8. Watch Mode
راجع قسم "التشغيل" أعلاه. تفصيل تقني: فحص دوري كل 3 ثوانٍ (قابل للتغيير عبر `WATCH_POLL_INTERVAL` في `config.py`) يبني "بصمة" لكل ملف صورة (المسار + وقت التعديل + الحجم)، وأي اختلاف يُحدِّث رندر الدفعة بالكامل تلقائياً.

### 9. Auto Backup
قبل بدء أي رندر، يُحفظ نسخة احتياطية (إعدادات + أي فريمات مرندرة جزئياً من جلسة سابقة منقطعة) في `backups/<اسم_المشروع>/`. لو انقطع البرنامج بشكل مفاجئ (انقطاع كهرباء، إغلاق قسري)، التشغيل التالي يكتشف ذلك ويعرض استعادته. تُحذف النسخة الاحتياطية تلقائياً بعد أي رندر ينجح بالكامل بلا أي فشل.

### 10. Render Statistics
تقرير كامل بعد كل رندر:
```
══════════════════════
   Render Complete
══════════════════════
  Frames        : ...
  Scenes        : ...
  FPS           : ...
  Resolution    : ...
  Output Size   : ... MB
  Time Taken    : ..:..
----------------------
  Avg/Frame     : ... ms
  AI Frames     : ...
══════════════════════
```

### 11. Logging System
كل جلسة تشغيل تُنشئ ملف سجل مستقل في `logs/` (يحتفظ بآخر 20 ملفاً تلقائياً)، يحتوي كل مراحل الرندر، التحذيرات، والأخطاء — مفيد جداً لتتبّع أي مشكلة لاحقاً.

### الميزات المحافَظ عليها من الإصدارات السابقة
Scene Sequencing، Metadata (MP4)، GIF متقدّم (ألوان/عرض/تكرار)، Compound Scenes، Exposure Sheet، Resume Rendering (استئناف رندر متوقَّف)، Preview، Batch Rendering — كل هذه تعمل بدون أي تغيير.

## أمثلة عملية

**رندر يدوي بسيط:**
```bash
python3 main.py
# اكتب مسار مجلد المشروع عند السؤال، واضغط Enter لقبول كل الافتراضيات
```

**مراقبة مستمرة لمشروع أثناء العمل عليه في FlipaClip:**
```bash
python3 main.py --watch /sdcard/MyAnimation

```

**استخدام إعدادات محفوظة سابقاً بسرعة:**
```bash
python3 main.py
# عند السؤال "Load saved project?" → yes
# عند السؤال "Use these settings directly?" → yes
```

## ملاحظات أداء على الهاتف
- ابدأ بـ `render_chunk_size` الافتراضي (50) — البرنامج يقترح قيمة مناسبة حسب RAM جهازك
- AI mode (خاصة `optical_flow` و `hybrid`) أبطأ من `linear`/`smart` — على هاتف ضعيف، فضّل الأوضاع الأسرع أو عطّل AI تماماً لمشاهد طويلة
- AI Cache (`ai_cache`) يُسرّع رندرات متكررة لنفس الفريمات — مفيد جداً مع Watch Mode
